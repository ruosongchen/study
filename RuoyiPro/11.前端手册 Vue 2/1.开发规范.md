# 开发规范

## [#](https://doc.iocoder.cn/vue2/dev-spec/#_1-view-页面)1. view 页面

在 [`@views` (opens new window)](https://github.com/YunaiV/ruoyi-vue-pro/tree/master/yudao-ui-admin/src/views)目录下，每个模块对应一个目录，它的所有功能的 `.vue` 都放在该目录里。

![ 目录](https://doc.iocoder.cn/img/Vue2/%E5%BC%80%E5%8F%91%E8%A7%84%E8%8C%83/01.png)

一般来说，一个路由对应一个 `.vue` 文件。

## [#](https://doc.iocoder.cn/vue2/dev-spec/#_2-api-请求)2. api 请求

在 [`@/api` (opens new window)](https://github.com/YunaiV/ruoyi-vue-pro/tree/master/yudao-ui-admin/src/api)目录下，每个模块对应一个 `.api` 文件。

![ 目录](https://doc.iocoder.cn/img/Vue2/%E5%BC%80%E5%8F%91%E8%A7%84%E8%8C%83/02.png)

每个 API 方法，会调用 `request` 方法，发起对后端 RESTful API 的调用。

### [#](https://doc.iocoder.cn/vue2/dev-spec/#_2-1-请求封装)2.1 请求封装

[`@/utils/request` (opens new window)](https://github.com/YunaiV/ruoyi-vue-pro/blob/master/yudao-ui-admin/src/utils/request.js)基于 [axios (opens new window)](http://axios-js.com/zh-cn/docs/index.html)封装，统一处理 GET、POST 方法的请求参数、请求头，以及错误提示信息等。

```javascript
// ① api/system/user.js
import request from '@/utils/request'

// 查询用户列表
export function listUser(query) {
    return request({
        url: '/system/user/page',
        method: 'get',
        params: query
    })
}

// ② views/system/user/index.vue
import { listUser } from "@/api/system/user";

export default {
  data() {
    userList: null,
    loading: true
  },
  methods: {
    getList() {
      this.loading = true
      listUser().then(response => {
        this.userList = response.rows
        this.loading = false
      })
    }
  }
}
```

如果想要自定义的 `baseURL` 基础路径，可以通过 `baseURL` 进行直接覆盖。示例如下：

```javascript
export function listUser(query) {
    return request({
        url: '/system/user/page',
        method: 'get',
        params: query,
        baseURL: 'https://www.iocoder.cn' // 自定义
    })
}
```
